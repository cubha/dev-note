## TODO-2: 항목/폴더의 폴더 간 이동 기능

**현재 상태**:
- DB 스키마: 폴더 `parentId`, 항목 `folderId`로 계층 구조 이미 지원 (폴더 안 폴더 가능)
- DnD: **같은 부모 내 순서 변경만 가능** (`activeItem.folderId !== overItem.folderId → return`)
- 폴더 간 이동, 항목을 다른 폴더로 이동하는 기능 없음

**구현 범위**:
- [ ] DnD로 항목을 폴더 위에 드롭 → `folderId` 변경 (폴더 간 이동)
- [ ] DnD로 폴더를 다른 폴더 위에 드롭 → `parentId` 변경 (폴더 중첩)
- [ ] DnD로 항목/폴더를 루트 영역에 드롭 → `folderId/parentId = null` (루트로 이동)
- [ ] 우클릭 메뉴에 "다른 폴더로 이동" 옵션 추가 (폴더 선택 서브메뉴)
- [ ] 순환 참조 방지: 폴더를 자기 하위 폴더로 이동 차단
- [ ] 다중 선택 + 폴더 이동 결합 (TODO-1 연계)

**설계 고려사항**:
- dnd-kit `onDragOver` 이벤트로 드롭 대상이 폴더인지 감지
- 드래그 중 폴더 위 호버 시 시각적 드롭존 표시 (하이라이트)
- 호버 일정 시간 유지 시 폴더 자동 펼침 (UX 개선, 선택적)
- 이동 시 `order` 값 재계산 필요 (대상 폴더 내 마지막 순서로 배치)

## TODO-3: 내보내기/가져오기 전략 재검토

**현재 상태**:
- 전체 DB 덤프 내보내기만 존재 (`exportData()` → 모든 folders + items JSON)
- 개별 항목/폴더 단위 내보내기 없음
- 가져오기는 Append(병합) 전략 — 기존 데이터 유지, 중복 검사 없음

**검토 포인트**:

### A. 전체 내보내기/가져오기 (현행 유지 + 개선)
- 현재 방식은 **백업/복원** 용도로 적합
- TODO-4와 연계하여 "전체 대체" 모드 추가 필요

### B. 개별 항목 내보내기 (신규 검토)
- **사용 시나리오**: 특정 SSH 설정, DB 접속정보 등을 팀원과 공유
- **고려사항**:
  - 암호화된 항목의 개별 내보내기 시 복호화 후 내보낼지, 암호화 상태 그대로 내보낼지
  - 암호화 상태 내보내기 → 같은 마스터 패스워드 사용자만 열 수 있음 (제한적)
  - 복호화 후 내보내기 → 평문 노출 위험 (보안 경고 필요)
  - 폴더 단위 내보내기 시 하위 항목 포함 여부
- **결론 보류**: 보안 정책과 실사용 시나리오에 따라 결정 필요

### C. 가져오기 모드 분리
- **Append (현행)**: 기존 데이터 + 새 데이터 병합 → 개별 가져오기에 적합
- **Replace (신규)**: 기존 데이터 전체 삭제 → 새 데이터로 대체 → 백업 복원에 적합
- 사용자에게 모드 선택 UI 제공 (모달 또는 드롭다운)

**결정 필요 사항**:
- [ ] 개별 내보내기 구현 여부 및 범위 (항목 단위? 폴더 단위?)
- [ ] 암호화 항목 개별 내보내기 시 복호화 정책
- [ ] 가져오기 모드 UI (Append/Replace 선택)
