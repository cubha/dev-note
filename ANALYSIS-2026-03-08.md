## 1. 프로젝트 개요

### 목적 및 핵심 가치
NDA 보안 환경에서 일하는 SI 프리랜서를 위한 **로컬 기반 접속/환경 정보 관리 도구**. 브라우저 IndexedDB만 사용하는 무설치 SPA로, 서버·DB·API·노트·마크다운 5종 카드 타입을 구조화하여 관리한다.

### 기술 스택

| 분류 | 기술 | 버전 |
|---|---|---|
| Framework | React 19 + Vite 7 | 19.2 / 7.3 |
| Language | TypeScript Strict | 5.9.3 |
| Data | Dexie.js v4 (IndexedDB) | 4.3 |
| State | Jotai + useLiveQuery | 2.18 / 4.2 |
| Editor | CodeMirror 6 | 6.x |
| Search | Fuse.js + Transformers.js | 7.1 / 3.8 |
| AI | Claude API (BYOK, 브라우저 직접 호출) | Haiku 4.5 |
| Styling | Tailwind CSS v4 | 4.2 |
| DnD | @dnd-kit | 6.3 / 10.0 |

### 현재 완성도
- **코어 CRUD**: 완성 (카드 생성/편집/삭제/정렬)
- **AI 기능**: 3계층 구현 (Smart Paste, 자연어 검색, 시맨틱 검색)
- **파일 I/O**: 내보내기/가져오기 완성 (v1/v2 호환)
- **코드 규모**: 38개 TS/TSX 파일, 약 6,800줄

## 2. 발견된 버그 근본 원인 분석

### Bug 1: 탭 포커스만으로 dirty 상태 활성화

**증상**: 수정하지 않고 Tab 키로 포커스만 이동해도 "수정됨" 표시가 나타남

**근본 원인**: `CardDetailEditor.tsx`의 CodeMirror `NoteEditor` 컴포넌트에서 **프로그래밍 방식(dispatch) 문서 교체가 onChange 콜백을 트리거**하는 구조적 결함

**상세 추적**:

```
[1] 탭 전환 발생: activeTab 변경
    ↓
[2] useEffect [activeTab] (60행): 상태 초기화, editorText = ''
    ↓
[3] useLiveQuery 완료: item 데이터 로드
    ↓
[4] useEffect [item] (71행):
    - initialLoadRef.current = true  ← 보호 시작
    - setEditorText(새 값)           ← React state 업데이트
    - initialLoadRef.current = false ← 보호 해제
    ↓
[5] NoteEditor useEffect [value] (526행):
    - view.dispatch({ changes: ... })  ← CodeMirror 문서 교체
    ↓
[6] EditorView.updateListener (467행):
    - update.docChanged === true  ← dispatch도 docChanged 트리거!
    - onChangeRef.current(text)   ← handleEditorChange 호출
    ↓
[7] handleEditorChange (133행):
    - markDirty() 호출
    - initialLoadRef.current === false ← 이미 보호 해제됨!
    - setDirty(true)  ← ❌ 실제 변경 없이 dirty!
```

**핵심 문제**: `initialLoadRef`가 [4]에서 동기적으로 false로 설정되지만, [5]의 useEffect는 React 렌더 사이클 이후에 실행되므로 [6] 시점에서는 보호가 해제된 상태. CodeMirror의 `view.dispatch`로 인한 `docChanged`를 사용자의 실제 입력과 구분할 수 없다.

**추가 경로**: `handleTypeChange` (139행) 호출 시에도 `markDirty()`가 먼저 호출된 후 `setFields()` 리빌드가 발생하여, 동일 타입 재선택 시에도 dirty가 설정됨.

### Bug 3: 카드 삭제 시 탭이 열리며 "불러오는 중..." 표시

**증상**: InfoCard의 `...` 메뉴에서 "삭제" 클릭 시, 해당 카드의 탭이 열리고 로딩 UI가 표시됨

**근본 원인**: `CardGrid.tsx`의 `handleDelete` 함수에서 **탭 상태 정리 없이 DB 삭제만 수행**

**상세 추적**:

```
[1] InfoCard 클릭 이벤트 전파:
    - InfoCard의 root div에 onClick={() => onEdit(item)} 바인딩 (62행)
    - 메뉴의 "삭제" 버튼은 e.stopPropagation() 미호출
    ↓
[2] 이벤트 버블링:
    - 삭제 MenuButton onClick → onDelete(item); setMenuOpen(false)
    - 이벤트가 부모 div의 onClick으로 버블링
    - onEdit(item) 실행 → openTab(item.id) 호출  ← ❌ 탭이 열림!
    ↓
[3] 탭 열림 + DB 삭제 경합:
    - openTab 실행: activeTab = item.id → CardDetailEditor 렌더링
    - handleDelete 실행: db.items.delete(item.id) → DB에서 삭제
    ↓
[4] CardDetailEditor 렌더링:
    - useLiveQuery → item === undefined → "불러오는 중..." 표시 (237행)
    - DB 삭제 완료 후 → item === null → "카드를 찾을 수 없습니다" (248행)
    - 하지만 탭은 openTabsAtom에 남아있음!
```

**이중 결함**:
1. InfoCard `onClick` 이벤트 버블링으로 삭제 시 `onEdit`이 동시 호출됨
2. `CardGrid.handleDelete` (105행)에서 탭/dirty 상태 정리를 하지 않음 (ContextMenu의 `removeItemsFromState` 패턴 미사용)

**동일 버그 존재 위치**: `useGlobalKeyboardShortcuts.ts` (98행)의 Delete 키 삭제도 탭/dirty 상태 정리 없이 `db.items.bulkDelete`만 호출

## 4. 신규 정보 구조(IA) 설계안

### 4.1 설계 철학

> **"자유형 문서 + AI 구조화 + 구조화 데이터 인라인 렌더링"**

레퍼런스 분석:
- **Notion**: 블록 기반 구조 → 유연하지만 접속 정보에 과도한 구조
- **Obsidian**: 순수 마크다운 + 링크 그래프 → 렌더링 자유도 높지만 구조화 부족
- **Bear**: 태그 기반 구분 + 마크다운 → 심플하지만 메타데이터 부족
- **Mem**: AI 네이티브 정리 → 입력 자유도 최고, 구조화를 AI에 위임

**dev-note 적합 모델**: `Bear의 심플함 + Notion의 블록 데이터 + Mem의 AI 정리`

### 4.2 데이터 모델 전환안

```
[현재]
Folder (1:N) → Item { title, type, content: StructuredContent }

[제안 — Phase별 점진 전환]

Phase A (현행 유지 + 버그 수정):
  Folder (1:N) → Item { title, type, content, tags }
  → 버그 수정만 수행, 구조 변경 없음

Phase B (자유형 콘텐츠 강화):
  Folder (1:N) → Item { title, type, content, tags }
  → type에 'document' 추가
  → content에 마크다운 + 구조화 블록 혼합 포맷 지원
  → AI가 자유 텍스트를 블록으로 변환

Phase C (블록 기반 문서):
  Folder (1:N) → Document { title, tags }
                    └─ Block[] { type, data, order }
  → 완전한 블록 기반 아키텍처
  → 각 블록: text, heading, credential, url, code, table 등
```

### 4.3 Phase B 상세 설계 (권장 — 최소 변경, 최대 효과)

**새로운 콘텐츠 포맷**:
```typescript
// 기존 StructuredContent 유지 + 새 HybridContent 추가
interface HybridContent {
  format: 'hybrid'
  sections: Section[]
}

interface Section {
  id: string           // nanoid
  type: 'markdown' | 'credentials' | 'urls' | 'keyvalue'
  title?: string       // 섹션 제목 (예: "AS-IS 환경", "접속 정보")
  data: SectionData    // 타입별 데이터
}

type SectionData =
  | { type: 'markdown'; text: string }
  | { type: 'credentials'; items: CredentialItem[] }
  | { type: 'urls'; items: UrlItem[] }
  | { type: 'keyvalue'; pairs: { key: string; value: string; secret?: boolean }[] }

interface CredentialItem {
  label: string         // "운영서버", "개발 DB"
  host: string
  port?: string
  username?: string
  password?: string     // 마스킹 렌더링
  note?: string
}

interface UrlItem {
  label: string
  url: string
  method?: string
  auth?: string
  note?: string
}
```

**DB 변경 (v10)**:
```typescript
// ItemType에 'document' 추가
type ItemType = 'server' | 'db' | 'api' | 'note' | 'custom' | 'document'

// content 필드는 이미 string(JSON)이므로 스키마 변경 불필요
// HybridContent를 serialize/parse할 수 있도록 content.ts 확장만 필요
```

**이점**:
- 기존 5종 카드 100% 호환 유지
- `document` 타입만 추가하여 자유형 문서 지원
- 하나의 문서 안에 여러 섹션(접속 정보 + URL + 메모) 혼합 가능
- AI가 자유 텍스트 → Section[] 변환 수행

## 5. AI 활용 흐름 재설계

### 5.1 Smart Paste 재설계 — "AI 구조화 엔진"

**현재**: 접속 정보 파싱기 (정규식 → AI 폴백)
**제안**: 자유형 텍스트를 의미 단위 섹션으로 자동 구조화하는 엔진

```
[새로운 흐름]

사용자 입력 (자유형 텍스트)
  │  예: "프로젝트A 운영 서버 10.0.0.1 포트22
  │       admin / password123
  │       DB는 mysql://10.0.0.2:3306/prod_db
  │       관리자 페이지 https://admin.example.com
  │       VPN 접속 후 사용할 것"
  │
  ▼
[Tier 1: 정규식 사전 분석 — 유지]
  - 접속 정보 패턴 감지 (host, URI, curl 등)
  - 감지 결과를 AI 프롬프트 힌트로 전달
  │
  ▼
[Tier 2: Claude API — 역할 확장]
  - 입력: 자유 텍스트 + Tier 1 힌트
  - 출력: Section[] (의미 단위 구조화)
  │
  ▼
[미리보기 + 편집]
  - 섹션별 미리보기 렌더링
  - 사용자가 섹션 순서/내용 수정 가능
  │
  ▼
[저장]
  - HybridContent로 직렬화 → DB 저장
```

**AI 프롬프트 설계 (Tier 2 확장)**:
```
You are a content organizer for a developer's project management tool.

Given free-form text, organize it into meaningful sections:
- Group related information (e.g., server access, DB connections, URLs)
- Detect credentials (host, port, username, password) and structure them
- Detect URLs and categorize them
- Keep unstructured notes as markdown sections
- Preserve all original information — do not remove anything

Output a JSON array of sections, each with:
- type: "credentials" | "urls" | "keyvalue" | "markdown"
- title: descriptive section title in Korean
- data: structured content matching the type
```

### 6.2 탭 구조 권장사항

**현행 유지 권장** (탭 + 카드 그리드 병행):
- 탭은 편집 시에만 사용 (현재와 동일)
- 카드 그리드는 전체 조회 용도 (현재와 동일)
- 플로팅 뷰(CardFloatingView)는 빠른 조회 용도 (현재와 동일)
- 탭 닫기 시 미저장 확인 다이얼로그 추가 (현재 누락)

**사이드바 개선**:
- 문서(document) 타입은 사이드바에서 직접 펼쳐보기 가능하도록
- 섹션 목록을 하위 트리로 표시 (접속 정보, URL, 메모 등)

### Bug 1 수정: 값 비교 기반 dirty 판단

**방향**: programmatic dispatch와 사용자 입력을 구분하는 ref 추가

```typescript
// CardDetailEditor.tsx — NoteEditor 컴포넌트

function NoteEditor({ value, placeholderText, onChange, onScroll }: Props) {
  const containerRef = useRef<HTMLDivElement>(null)
  const viewRef = useRef<EditorView | null>(null)
  const onChangeRef = useRef(onChange)
  const onScrollRef = useRef(onScroll)
  const isProgrammaticRef = useRef(false)  // ← 추가
  const placeholderCompartment = useRef(new Compartment())

  // 마운트
  useEffect(() => {
    if (!containerRef.current) return
    const view = new EditorView({
      state: CMState.create({
        doc: value,
        extensions: [
          // ... 기존 extensions ...
          EditorView.updateListener.of((update) => {
            if (update.docChanged && !isProgrammaticRef.current) {  // ← 변경
              onChangeRef.current(update.state.doc.toString())
            }
          }),
          // ...
        ],
      }),
      parent: containerRef.current,
    })
    viewRef.current = view
    return () => { view.destroy(); viewRef.current = null }
  }, [])

  // 외부 value 변경 시
  useEffect(() => {
    const view = viewRef.current
    if (!view) return
    const current = view.state.doc.toString()
    if (current !== value) {
      isProgrammaticRef.current = true  // ← 보호 시작
      view.dispatch({
        changes: { from: 0, to: current.length, insert: value },
      })
      isProgrammaticRef.current = false  // ← 보호 해제
    }
  }, [value])
}
```

```typescript
// src/store/tabHelpers.ts — 공용 삭제 상태 정리 함수 추가

export function removeItemsFromState(
  ids: number[],
  setOpenTabs: Setter<number[]>,
  setActiveTab: Setter<number | null>,
  setDirtyItems: Setter<Set<number>>,
) {
  setOpenTabs((prev) => prev.filter((id) => !ids.includes(id)))
  setActiveTab((prev) =>
    prev !== null && ids.includes(prev) ? null : prev,
  )
  setDirtyItems((prev) => {
    const next = new Set(prev)
    ids.forEach((id) => next.delete(id))
    return next
  })
}
```

}
